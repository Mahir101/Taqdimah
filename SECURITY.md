# Security Policy

Taqdimah will handle sensitive community data, including trust records, organization data, welfare workflows, and potentially zakat-related information. Security and privacy must be treated as core product requirements.

## Reporting a Vulnerability

Do not disclose vulnerabilities publicly.

Preferred reporting path:

1. Use GitHub private vulnerability reporting if it is enabled for this repository.
2. If it is not enabled, contact a maintainer privately.
3. If no private contact is available, open a minimal public issue titled `Security contact needed` and do not include technical details.

Please include, privately:

- A short description of the issue.
- Steps to reproduce.
- Potential impact.
- Affected files, docs, endpoints, or workflows.
- Any suggested fix, if available.

## Sensitive Data

Never post these publicly:

- Personal hardship cases.
- Zakat or welfare recipient details.
- Identity documents.
- Addresses, phone numbers, or private financial records.
- Authentication tokens, secrets, API keys, or credentials.
- Private organization records.

## Security Priorities

- Protect people seeking aid.
- Protect organization and donor records.
- Prevent abuse of trust, ranking, and verification systems.
- Keep audit trails for sensitive workflows.
- Minimize collected data.
- Restrict access by role.
- Prefer simple, reviewable security controls.

## Supported Versions

Taqdimah is currently pre-MVP. Security fixes should target the active development branch and documentation that defines future implementation.
