# Open Source Readiness Checklist

This checklist tracks what Taqdimah should have before presenting itself as a mature open source project.

## Added

- [x] README
- [x] Code of Conduct
- [x] Contributing Guide
- [x] Security Policy
- [x] Support Guide
- [x] Issue templates
- [x] Discussion templates
- [x] Pull request template

## Still Needed

- [ ] Choose a license.
- [ ] Add `LICENSE`.
- [ ] Decide maintainer contacts for conduct and security reports.
- [ ] Enable GitHub Discussions.
- [ ] Enable private vulnerability reporting if available.
- [ ] Add project labels.
- [ ] Add a public roadmap or keep `docs/TASKS.md` as the roadmap.
- [ ] Add setup instructions once code exists.
- [ ] Add tests and CI once code exists.
- [ ] Add release notes once versions exist.

## License Options to Consider

- `MIT`: simple and permissive.
- `Apache-2.0`: permissive with explicit patent terms.
- `GPL-3.0`: requires derivative works to stay open under the same license.
- `AGPL-3.0`: like GPL, but also covers network/server use.

Choose intentionally. A license changes what others are allowed to do with the project.
