# Contributing to Taqdimah

Thank you for considering a contribution. Taqdimah is an early-stage public-benefit project focused on tazkiyah, trusted discovery, zakat and welfare coordination, Islamic organization partnerships, conflict resolution, and halal economic development.

Contributions should help the Ummah with sincerity, dignity, and practical usefulness.

## Ways to Contribute

- Improve documentation, diagrams, and product thinking.
- Review flows for zakat, sadaqah, welfare, privacy, and organization coordination.
- Suggest features that reduce suffering and strengthen communities.
- Help with UX, accessibility, localization, and Bengali/English content.
- Identify risks in trust, verification, ranking, privacy, and moderation.
- Build implementation tasks once code begins.
- Share research from Islamic public-benefit tech, zakat systems, masjid operations, and ethical economic development.

## Contribution Rules

- Keep changes focused and reviewable.
- Do one meaningful task per pull request.
- Do not add monetization, ranking, or welfare-flow changes without explaining the ethics and trust impact.
- Do not add features that expose private hardship, zakat, family, legal, or financial information.
- Do not present religious, legal, tax, medical, or financial claims as authoritative unless qualified review is part of the work.
- Prefer simple, auditable systems over opaque automation.
- Keep the project low-burden for small vendors, aid seekers, and community organizations.

## Issues and Discussions

Use Discussions for questions, ideas, community feedback, and early product exploration.

Use Issues for specific work:

- Bugs in documentation or code.
- Clear feature proposals.
- Missing requirements.
- Security-safe technical tasks.
- Documentation improvements.

Do not use public issues for private welfare cases or security vulnerabilities.

## Pull Request Process

1. Open or find an issue/discussion for the work when useful.
2. Keep the change scoped.
3. Update related documentation.
4. Add tests when code exists and the change affects behavior.
5. Fill out the pull request template.
6. Wait for maintainer review before merging.

## Commit Style

Use clear commit messages:

```text
docs: add code of conduct
docs: clarify zakat workflow goals
feat: add organization profile model
fix: correct trust score example
```

## Review Priorities

Maintainers will review for:

- Benefit to the Ummah.
- Shariah and ethics risk.
- Privacy and dignity.
- Trust and verification impact.
- Simplicity and maintainability.
- Consistency with the existing docs.
